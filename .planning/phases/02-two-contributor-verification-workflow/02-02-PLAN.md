Named tunable constants (D-08) — pick launch-safe defaults, document rationale:
ACCOUNT_AGE_SOFT_CUTOFF_HOURS=24, CONFIRMATION_COOLDOWN_WINDOW_HOURS=1, CONFIRMATION_MAX_PER_WINDOW=5,
CONFIRMATION_MAX_PER_DAY=20, YOUNG_ACCOUNT_PENALTY=-1, ESTABLISHED_ACCOUNT_BONUS=+1,
SAME_SUBNET_PENALTY=-1, DISTINCT_SUBNET_BONUS=+1, TRUST_BLOCK_THRESHOLD=-2, IPV4_PREFIX=24, IPV6_PREFIX=48.

Trust-score rule (weighted, offsetting, fail-open): start score=0. Account-age known & < cutoff →
+YOUNG_ACCOUNT_PENALTY; known & >= cutoff → +ESTABLISHED_ACCOUNT_BONUS. IP known & equal to
submitter subnet hash → +SAME_SUBNET_PENALTY; known & distinct → +DISTINCT_SUBNET_BONUS; IP or
submitter-hash absent → 0 (fail-open). `trust_ok = score > TRUST_BLOCK_THRESHOLD`. This blocks
ONLY the exact Sybil signature (fresh account AND same subnet = -2); young-alone (-1) passes, so
account-age is never a hard reject on its own (D-08).

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| confirmer request → API | Client IP arrives here (possibly via proxy / spoofable XFF); it is used only as a SOFT signal and pseudonymized before storage. |
| API → Postgres disc_edits | The confirmation counter and IP-hash persist here — the single shared source of truth across all gunicorn workers. |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-2-02 | Spoofing / Elevation | Sybil farm of fresh accounts rapidly confirming | high | mitigate | Postgres confirmation cooldown hard floor (CONFIRMATION_MAX_PER_WINDOW/PER_DAY) + fresh-account soft penalty (D-13/D-08). |
| T-2-03 | Spoofing | Same-operator confirmer on the same /24 | medium | mitigate | IP-diversity soft penalty via salted /24 hash; offsettable, fail-open (D-04/D-06). |
| T-2-04 | Tampering | X-Forwarded-For spoofing to fake IP diversity | medium | mitigate | IP is a SOFT positive-only signal; a spoof only removes a penalty (looks diverse) and never triggers a hard block; cooldown stays intact (Pitfall 2). |
| T-2-07 | Information Disclosure | Raw IP leaked via DB or logs | high | mitigate | Store only salted, /24-truncated HMAC-SHA256; never persist or log raw IP; nullable column, bounded retention documented in Plan 05 (D-06). |
| T-2-08 | Elevation | Cooldown bypass via multi-worker in-memory limiter | high | mitigate | Counter is a Postgres COUNT over disc_edits, NOT the Nx-inflated in-memory slowapi limiter (D-13/D-14). |
| T-2-SC | Tampering | package installs | low | accept | No new packages — stdlib `ipaddress`/`hmac`/`hashlib`/`datetime` only (RESEARCH audit = none). |

No high-severity threat is left unmitigated by this plan.
</threat_model>
