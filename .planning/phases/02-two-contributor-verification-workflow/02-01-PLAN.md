<objective>
Create `api/app/structural_match.py`: a tolerant, canonical structural-equality comparison
between a submitted `DiscSubmitRequest` payload and the STORED `DiscTitle`/`DiscTrack` rows of
an existing disc (D-03). This is the proof-of-possession gate — a second contributor can only
verify a disc by reproducing structure that is withheld from public reads (D-08/D-09), so the
match must prove they read a physical disc, not that they echoed public metadata.

Purpose: Upgrade the verify trigger from public release fields (`_releases_match` — title/year/
tmdb_id, all publicly searchable) to withheld structure. This is the single highest-risk design
decision in the phase (RESEARCH "Key insight"): the tolerance envelope must accept benign
independent-rip jitter yet still catch real structural differences.
Output: `structural_match.py` plus `test_structural_match.py` pinning the verify/dispute boundary
in both directions.
</objective>
