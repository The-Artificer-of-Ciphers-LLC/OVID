<objective>
Withhold the full submitted structural payload of an `unverified` disc from public reads until it
is verified (phase success criterion 4, D-09/D-11/D-12). Implement a single status branch in
`_disc_to_response` that omits `titles` (and their chapters/main-feature marker/audio-subtitle
tracks) when `disc.status == "unverified"`, while always returning `release` and
`fingerprint_aliases`. Confirm/annotate that `DiscLookupResponse`'s structural fields are already
optional so an empty `titles` list is a valid, non-breaking response.

Purpose: This is the anti-echo defense — it forces a would-be confirmer to compute the structure
from a real physical disc rather than copying the first submitter's upload. It is the read-side
counterpart to the structural-match verify gate (Plan 01/03): the confirmer must reproduce exactly
the data this redaction hides. Redaction is uniform for all readers (no optional-auth path on the
cacheable anonymous GET — submitter-preview is explicitly deferred, D-12).
Output: modified _disc_to_response, a documenting schema note, and test_lookup_redaction.py.
</objective>
