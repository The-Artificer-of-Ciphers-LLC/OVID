<objective>
Wire the two Wave-1 primitives into the confirmation path and remove the no-proof verify route:
in `_handle_existing_disc` (routes/disc.py), insert the anti-Sybil gate as a pre-check (cooldown →
429, soft-score → 403), replace the public-field `_releases_match` verify trigger with
`structural_match` (D-01/D-03), retain release-consistency as the DISPUTE trigger (Open Question
A3), capture `ip_hash` on the confirmation `DiscEdit`, and store `ip_hash` on the initial create
edit so future confirmers can be compared. Then DELETE the bodyless `POST /v1/disc/{fingerprint}/verify`
route and its entire test file (D-02) — the pure Sybil bypass with no legitimate caller.

Purpose: Make the two-contributor trust model live end-to-end — confirmation now requires proof of
physical possession (reproducing withheld structure) gated by anti-Sybil weighting, and the weakest
link (a status flip on a bare bearer token) is gone. All status writes stay in `verification.py`
(VERIFY-02) — the gate and match wrap the primitives, never mutate `disc.status` directly.
Output: modified routes/disc.py, new confirmation-flow + route-retired tests, deleted verify tests.
</objective>

Pinned design decisions (make visible in the code + tests):
- OQ1/A3: structural equality gates VERIFY; release-consistency (`_releases_match`) is retained as
  the DISPUTE trigger. Truth table: structural-match + release-match → verify; structural-match +
  release-mismatch → dispute; structural-mismatch → dispute (A2 keeps a verified disc verified).
- OQ4: cooldown hard-block → 429 `rate_limited` with a `Retry-After` header; soft-score reject →
  403 `insufficient_trust`. Both keep the `{request_id, error, message}` envelope + x-request-id.
- `_handle_existing_disc` gains a `request: Request` parameter (threaded from both call sites in
  submit_disc: the up-front duplicate check ~line 733 and the losing-race recovery ~line 794).

| retired /verify surface | Removed — no bodyless status-flip endpoint remains. |

| T-2-06 | Tampering | Third party silently flips a verified disc to disputed | high | mitigate | flag_dispute() no-ops on verified; A2 dispute_attempted "stays verified" 200 branch preserved (VERIFY-03). |
| T-2-09 | Elevation | No-proof POST /{fp}/verify endpoint as a pure Sybil bypass | high | mitigate | Route + test file DELETED (D-02); the only confirmation path now requires proof-of-possession + anti-Sybil gate. |
