<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| container network → redis | The rate-limit store holds ephemeral counters; if reachable from outside the compose network it becomes tamper/DoS surface |
| operator → env config | `REDIS_URL`/`OVID_WORKERS` in compose/.env drive the correctness of the control; a wrong value weakens or disables it |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-03-05 | Information Disclosure / Tampering | redis service exposed on an untrusted network | medium | mitigate | Internal-only service, NO published host port in prod (mirrors prod `db`); ephemeral, no persistence; if ever externalized, use `rediss://` + AUTH (RESEARCH A4) — documented |
| T-03-06 | Denial of Service (config bypass) | multi-worker deploy shipped without `REDIS_URL` | high | mitigate | Compose passes `OVID_WORKERS=4` + `REDIS_URL` together; the Plan 01 D-06 guard refuses to boot if they are ever separated; documented in .env.example + deployment docs |
| T-03-07 | Information Disclosure | secrets committed via `.env.example` | low | accept | `.env.example` carries placeholders only (`REDIS_URL: redis://redis:6379/0` is an internal non-secret address); real `.env` is git-ignored (existing convention) |
</threat_model>
