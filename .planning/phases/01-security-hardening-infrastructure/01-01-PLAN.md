```python
def _validate_secret_key(key: str) -> str:
    """Validate JWT signing key strength. Fails fast if too short."""
    if len(key.encode("utf-8")) < 32:
        raise RuntimeError(
            f"OVID_SECRET_KEY is too short ({len(key)} bytes). "
            "Must be at least 32 bytes for HS256 signing security. "
            "Generate with: python3 -c \"import secrets; print(secrets.token_urlsafe(48))\""
        )
    return key

def _validate_apple_key_if_set() -> None:
    """Validate Apple Sign-In private key if configured. Skip if unset."""
    raw = os.environ.get("APPLE_PRIVATE_KEY", "")
    if not raw:
        return  # Not configured — provider just won't appear
    import base64
    from cryptography.hazmat.primitives.serialization import load_pem_private_key
    if "BEGIN" not in raw:
        try:
            raw = base64.b64decode(raw).decode("utf-8")
        except Exception:
            raise RuntimeError(
                "APPLE_PRIVATE_KEY is set but is not valid base64 or PEM. "
                "Provide the ES256 private key as PEM text or base64-encoded PEM."
            )
    try:
        load_pem_private_key(raw.encode("utf-8"), password=None)
    except Exception as e:
        raise RuntimeError(
            f"APPLE_PRIVATE_KEY is set but cannot be loaded as an EC private key: {e}"
        )
```

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| env vars -> app config | Secrets loaded from environment at startup |
| Redis client -> Valkey server | Internal Docker network, no auth by default |
| rate limiter -> Redis storage | Rate counters stored in shared Redis |
| session middleware -> Redis | Session data stored server-side |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-1-01 | Denial of Service | Redis unavailable crashes API | mitigate | `api/app/redis.py` returns None on failure; rate limiter falls back to memory://; session middleware falls back to empty session (D-17, INFRA-03) |
| T-1-02 | Information Disclosure | Redis URL with password logged | mitigate | `init_redis()` logs only host portion: `redis_url.split("@")[-1]` |
| T-1-03 | Tampering | Weak JWT secret allows token forgery | mitigate | `_validate_secret_key()` rejects keys shorter than 32 bytes at startup (SEC-03) |
| T-1-04 | Information Disclosure | Apple private key misconfiguration leaks error | mitigate | `_validate_apple_key_if_set()` catches at startup, RuntimeError has safe message (SEC-04) |
| T-1-05 | Elevation of Privilege | Session cookie without HttpOnly | mitigate | `RedisSessionMiddleware` sets HttpOnly=True, SameSite=Lax on session cookie |
| T-1-06 | Spoofing | Session secret same as JWT secret | mitigate | Separate `SESSION_SECRET_KEY` env var (D-06), falls back to OVID_SECRET_KEY only as migration path |
</threat_model>
