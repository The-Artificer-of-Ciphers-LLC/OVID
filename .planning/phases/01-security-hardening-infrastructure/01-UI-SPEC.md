## Applicability

**This phase has no UI contract.** Phase 1 (Security Hardening & Infrastructure) is entirely backend work: OAuth auth flow rewrite, Valkey/Redis infrastructure, Mastodon security hardening, startup validation, and bug fixes (SEC-01 through SEC-06, BUG-01 through BUG-05, INFRA-01 through INFRA-04).

The ROADMAP does not mark Phase 1 with a UI hint. No requirements in this phase involve visual components, user-facing pages, or interaction patterns.

### Frontend-Adjacent Changes (No Visual Impact)

- **D-03 (Cookie auth):** Web client switches from localStorage JWT to HttpOnly cookie + flag cookie. This changes `lib/auth.ts` plumbing (how `useAuth()` detects login state) but introduces zero visual changes. No new UI elements, no changed layouts, no user-visible state transitions.
- **D-09 (CORS):** `ALLOWED_ORIGINS` env var configuration. Server-side only.

These are implementation details consumed by the executor, not design contracts.
