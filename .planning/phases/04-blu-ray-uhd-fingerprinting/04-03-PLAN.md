<objective>
Remove `BDDisc._build()`'s Tier-1-short-circuit (the FPRINT-03 core gap at `bd_disc.py:114-133`) and delegate identity construction to `identify_bd()` (Plan 04-02). Consolidate `BDDisc.playlists`' filtering to use the same `select_canonical_playlists()` survivor set that produces the Tier-2 fingerprint (Plan 04-01), so decoy/obfuscation playlists are excluded from the submitted title list, not just the hash. Expose the full alias pair via a new `.identity` property while keeping `.fingerprint`/`.tier` as back-compat proxies computed from `.identity.primary` (D-04). Rewrite the three existing `test_bd_disc.py` tests that assert the old short-circuit behavior, and add new tests for the alias-pair, diagnostics, and playlist-consolidation guarantees.

Purpose: This is where FPRINT-03 actually lands end-to-end. `test_bd_disc.py` already has extensive `BDDisc` integration coverage (24KB, 8 test classes) written against the CURRENT short-circuit behavior — three of its existing tests (`test_tier1_with_aacs`, `test_tier1_uhd`, `test_cli_fingerprint_bd_tier1`) assert `disc.tier == 1`/`disc.fingerprint.startswith("bd1-aacs-")` when AACS is present, which becomes incorrect once Tier-2 is unconditionally primary. One existing test (`test_all_playlists_under_60s_with_aacs_uses_tier1`) encodes a real degenerate-case regression guard that Plan 04-02's `identify_bd()` was specifically designed to preserve — this plan proves that end-to-end.
Output: A refactored `bd_disc.py` with `.identity`/`_identity_set`, and `test_bd_disc.py` updated to assert the new alias-pair behavior without losing any existing regression coverage.
</objective>
