<objective>
Add `identify_bd()` to `disc_identity.py`, mirroring the existing `identify_dvd()` pattern exactly: Tier-2 (BDMV structure) is attempted first and used as primary whenever computable, Tier-1 (AACS) is attempted independently and attached as an alias, and every branch records a diagnostic. Unlike a pure DVD mirror, this function must also handle the genuinely degenerate case where Tier-2 itself cannot be computed (zero playlists survive the anti-obfuscation filter) — in that case, and only that case, Tier-1 becomes primary as a documented fallback, preserving a real regression guarantee already encoded in `ovid-client/tests/test_bd_disc.py::test_all_playlists_under_60s_with_aacs_uses_tier1` (a disc with only short playlists but a readable AACS directory must still produce a fingerprint).

Purpose: This is the FPRINT-03 gap-closing seam (D-02, D-03, D-04) — it stops `BDDisc` from ever short-circuiting on Tier-1 success while preserving the disc's only-available identity in the fully-degenerate case, matching D-01's "Tier-2 primary when computable" framing without regressing the existing all-short-playlists-with-AACS guarantee.
Output: `identify_bd()` plus its two identity-builder helpers and method/version constants in `disc_identity.py`, and a new `test_bd_identity.py` mirroring `test_disc_identity.py`'s 5-test shape (expanded to 8 to cover the Tier-2-unavailable fallback and hash-failure paths).
</objective>

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-04-04 | Tampering | `identify_bd()` Tier-2-vs-Tier-1 primary selection | high | mitigate | Tier-2 (structural) is unconditionally attempted and used as primary whenever computable; Tier-1 is appended only as an alias — closes the "AACS-stripped rip silently reports a different fingerprint namespace" gap (FPRINT-03, D-01/D-02). | T-04-05 | Information Disclosure | `aacs_identity()` / `compute_aacs_fingerprint()` call site inside `identify_bd()` | high | mitigate | `identify_bd()` only ever passes the raw `Unit_Key_RO.inf` bytes into a one-way SHA-1 hash; no code path stores, logs, or returns `unit_key_data` itself — only the derived `DiscIdentity.fingerprint` string leaves the function (D-11). |
| T-04-06 | Denial of Service | `identify_bd()` exception handling around `aacs_identity()` | medium | mitigate | The `try/except Exception` around the Tier-1 attempt records an `aacs_fingerprint_failed` diagnostic instead of propagating — a malformed or unexpected `Unit_Key_RO.inf` payload can't crash `identify_bd()`. |
