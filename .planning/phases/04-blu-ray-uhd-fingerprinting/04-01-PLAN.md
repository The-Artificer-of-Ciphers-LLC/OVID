<objective>
Freeze the OVID-BD-2 Tier-2 anti-obfuscation ruleset (FPRINT-06, D-06/D-07/D-08) into a dedicated `bd2_spec.py` constants module, and replace `bd_fingerprint.py`'s filename-based tie-break with the locked content-based filter → max-clip-repeat filter → clip-sequence dedup → clip-sequence tie-break pipeline. Also apply the FPRINT-01/D-10 docstring wording fix ("AACS Disc ID" ≡ `SHA-1(AACS/Unit_Key_RO.inf)`) while this file is already open for edit.

Purpose: The current `filename ASC` tie-break at `bd_fingerprint.py:101` is a real, exploited obfuscation hole (studios renumber `.mpls` files — Lionsgate "ScreenPass" and similar decoy schemes). D-06 locks the replacement ruleset; D-07 requires it to live in a frozen, version-tagged module so a future accidental constant edit fails CI instead of silently fragmenting the fingerprint space. This is also the foundation Plan 04-03 (BDDisc refactor) and Plan 04-05 (pinned/obfuscated fixture corpus) build on.
Output: `ovid-client/src/ovid/bd2_spec.py` (new), an updated `build_bd_canonical_string()` in `bd_fingerprint.py` that filters/dedups/sorts via the frozen constants, and extended/corrected tests in `test_bd_fingerprint.py`.
</objective>
