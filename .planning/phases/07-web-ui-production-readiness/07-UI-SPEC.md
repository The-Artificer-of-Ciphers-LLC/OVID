# Phase 7 — UI Design Contract

> Visual and interaction contract for frontend phases. Generated by gsd-ui-researcher, verified by gsd-ui-checker.
>
> **Scope note:** This phase is production polish + accessibility + completeness of a mostly-existing
> Next.js 16 bespoke-Tailwind UI, NOT greenfield. This contract formalizes the design tokens the
> existing UI already implies (D-02) and raises every interactive element to the D-03 accessibility
> floor. Where "existing" is noted, the value is descriptive of current code; where "target" is noted,
> the value is the prescriptive contract for the `@theme` token layer + shared primitives (Button,
> Input, Field) this phase introduces. Migration is incremental — no re-platform, no component library,
> no new runtime deps.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | none (shadcn/ui explicitly deferred per D-02) |
| Preset | not applicable |
| Component library | none — bespoke Tailwind v4 + a small shared-primitive set (Button, Input, Field) introduced this phase |
| Icon library | none (no new runtime deps per D-02) — Unicode glyphs (`◉`, `←`/`→`) + emoji for provider marks; use inline SVG for any new icon, never an icon package |
| Font | Geist Sans (`--font-geist-sans`) for UI/body; Geist Mono (`--font-geist-mono`) for fingerprint / identity strings and any monospace code display |

**Token layer (D-02):** Introduce a Tailwind v4 `@theme` block defining the color / spacing / radius
tokens below, with dark mode via `@custom-variant dark`. Replace the current ad-hoc
`:root` + `@media (prefers-color-scheme: dark)` block in `web/app/globals.css`.

**Drift to fix (in scope):** `web/app/globals.css` still carries the Next.js starter override
`body { font-family: Arial, Helvetica, sans-serif; }` which contradicts the `font-sans` (Geist) class
applied in `layout.tsx`. Remove it so Geist is the single source of truth for the sans stack.

## Visual Hierarchy

One clear focal point per primary surface — the accent (10%) and Display/Heading type steer the eye there first.

| Surface | Focal point (draws the eye first) |
|---------|-----------------------------------|
| Search (`page.tsx`) | The **search input** is the primary anchor — centered, widest control, accent "Search discs" CTA; results list is secondary and only appears after query. |
| Disc detail (`disc/[fingerprint]/page.tsx`) | The **title + primary-fingerprint block** at the top — Display title over the Geist Mono canonical fingerprint; aliases, chapters, and tracks read as supporting detail below. |
| Submit (`submit/page.tsx`) | The form's **first field** — a single-column form with one visible primary "Submit disc" CTA; the eye enters at the top field and flows straight down. |
| Settings (`settings/page.tsx`) | The **linked-providers list** — the account's active login methods are the anchor; add/remove controls and any conflict banner orbit it. |

## Spacing Scale

Tailwind-native 4px base scale (already in use throughout the app). Declared canonical tokens:

Every token is a multiple of 4 — **zero sub-4px values**, no exceptions.

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon/inline gaps (`gap-1`), tight control padding, badge vertical padding (`py-1`) |
| sm | 8px | Compact spacing, input vertical padding (`py-2`), badge horizontal padding |
| — | 12px | Row rhythm (`gap-3`, `py-3`) — list rows, form field gaps (multiple of 4, retained) |
| md | 16px | Default element spacing, page gutter (`px-4`), card padding (`p-4`) |
| lg | 24px | Section padding (`py-6` pagination block) |
| xl | 32px | Layout / vertical section gaps (`py-8`, `mb-8`) |
| 2xl | 48px | Major section breaks |
| 3xl | 64px | Page-level spacing |

Formerly-sub-4px usages are removed: badge vertical padding moves `py-0.5` → `py-1` (4px), and the
`mt-0.5` (2px) text nudge on `DiscCard` / status badges is dropped (use 0, or `mt-1` = 4px if a gap is
needed). Tighter inline-pill rhythm, if genuinely required, is achieved via `line-height` / `text-*`
sizing — never a sub-4px spacing token. Do not introduce any sub-4px spacing anywhere.

Radius: `rounded` (4px) is the default control radius; `rounded-lg` (8px) for cards. Declare as
tokens `--radius-control: 4px` / `--radius-card: 8px`.

## Typography

Geist Sans throughout; Geist Mono for fingerprint / identity strings. Line heights are prescriptive
targets for the token layer.

Exactly **four sizes** and exactly **two weights** — no exceptions.

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Display (page h1 / brand) | 36px (`text-4xl`) | 600 semibold | 1.1 |
| Heading (section h2 / card h3) | 20px (`text-xl`) | 600 semibold | 1.2 |
| Body | 16px (`text-base`) | 400 regular | 1.5 |
| Label / meta / control | 14px (`text-sm`) | 600 (labels/small buttons) · 400 (meta) | 1.4 |

Notes:
- The **only** two weights are **400 regular + 600 semibold**. Form labels and small action buttons
  use 600 (formerly 500). The display h1 and brand mark use 600 (formerly 700). Do not add `500`,
  `700`, or any further weight.
- There is **no 12px tier**. Former `text-xs` (12px) badge / small-button text folds UP to the 14px
  label/meta size (`text-sm`) — 14px is the smallest type in the system, never smaller (WCAG legibility).
- Fingerprint / disc-identity strings and per-title chapter timing render in Geist Mono at the Body
  or label/meta size so long hashes stay scannable.

## Color

60/30/10 split. Hex values are the canonical tokens for both themes (`@theme` + `@custom-variant dark`).

| Role | Value (light / dark) | Usage |
|------|----------------------|-------|
| Dominant (60%) | `#ffffff` / `#0a0a0a` (fg `#171717` / `#ededed`) | Page background, primary surfaces |
| Secondary (30%) | `neutral-50/100` fills · `neutral-200`/`neutral-800` borders · `neutral-500` muted text | Cards, nav bar, list dividers, secondary/ghost buttons, muted/meta text |
| Accent (10%) | `#2563eb` (blue-600); ring `#3b82f6` (blue-500); hover `#1d4ed8` (blue-700) | See reserved list below |
| Destructive | `#dc2626` (red-600) family; banner `red-50/red-200/red-700` light, `red-950/red-800/red-300` dark | Error banners, destructive confirm actions only |

Accent reserved for: **primary action buttons** (Search discs, Submit disc, Login), the **active/hover link
state**, the **`:focus-visible` ring on every interactive element**, and the **brand mark** (`◉ OVID`).
Never apply accent to secondary/ghost buttons (Unlink, Previous/Next, Logout) — those stay
neutral-bordered.

Semantic status palette (functional, distinct from the 10% accent — do not expand ad hoc):

| Status | Light | Dark |
|--------|-------|------|
| Movie / info | `bg-blue-100 text-blue-800` | `bg-blue-900 text-blue-200` |
| TV | `bg-purple-100 text-purple-800` | `bg-purple-900 text-purple-200` |
| Documentary / verified-ok | `bg-green-100 text-green-800` | `bg-green-900 text-green-200` |
| Unverified / disputed (recommend) | `bg-amber-100 text-amber-800` | `bg-amber-900 text-amber-200` |
| Neutral / unknown | `bg-neutral-100 text-neutral-700` | `bg-neutral-800 text-neutral-300` |

**D-03 contrast gate (blocking):** Every text/UI pairing above MUST be verified at WCAG AA
(4.5:1 text, 3:1 UI/border) in BOTH light and dark before sign-off. Known review targets:
`neutral-400` on white (used for empty-state hints and the nav "…" loader) is borderline — verify or
darken to `neutral-500`. The blue-600-on-white primary button and blue-500 focus ring must clear 3:1
against their adjacent surfaces in both themes.

**Focus (D-03, blocking):** Current inputs use `focus:` (fires on mouse focus) and buttons have NO
focus ring. The shared primitives and every interactive element MUST adopt a visible
`focus-visible:` ring (`focus-visible:ring-2 focus-visible:ring-blue-500 focus-visible:ring-offset-2`
with a dark-mode offset color) — keyboard-visible, not mouse-noisy.

## Copywriting Contract

Representative canonical copy across the phase's four surfaces (search, disc detail, submit, settings).
Existing copy is kept where it already reads well; new copy is prescriptive.

| Element | Copy |
|---------|------|
| Primary CTA — search | "Search discs" |
| Primary CTA — submit flow | "Submit disc" |
| Primary CTA — settings link-provider | "Link a provider" |
| Empty state — no search yet (heading/body) | (centered hint) "Enter a title to search the database." |
| Empty state — zero results | "No releases found." — followed by "Check the spelling or try a broader title." |
| Empty state — disc detail no aliases/chapters | "No additional fingerprint aliases recorded." / "No chapters recorded for this title." |
| Error state — search/API failure | Red banner: the API error message + "Please try again." (parse/validation errors surface via `data-testid="parse-error"` per project convention) |
| Destructive confirmation — unlink provider | Unlink `{Provider}`: on the last remaining method the button is disabled; if the API returns `400 cannot_unlink_last`, show "You must keep at least one login method. Link another provider before removing this one." |
| Merge / email-conflict banner (D-05, enumeration-safe) | "This email is already linked to another OVID account via **{already-linked provider(s)}**. To connect this login, re-authenticate with that provider." + a direct re-auth link (`GET /v1/auth/<provider>/login?pending_link_id=<id>`). MUST name only the provider(s) already on the *current* account context — never reveal the matched account, email, or that a match on a *different* account exists (mirror ME-02 guard). |

Voice: concise, imperative, no exclamation marks; error copy always states the problem AND the next
step. `aria-live="polite"` on validation-error and banner regions (D-03).
