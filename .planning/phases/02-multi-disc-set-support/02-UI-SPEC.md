# Phase 2 — UI Design Contract

> Visual and interaction contract for multi-disc set support UI components. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

## Design System

| Property | Value |
|----------|-------|
| Tool | none |
| Preset | not applicable |
| Component library | none (hand-written Tailwind components) |

## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline padding (`gap-1`, `py-1`, `px-1`) |
| sm | 8px | Compact element spacing (`gap-2`, `py-2`, `mb-2`, `px-2`) |
| md | 16px | Default element spacing (`gap-4`, `px-4`, `mb-4`, `space-y-4`) |
| lg | 24px | Section padding (`mb-6`, `py-6`) |
| xl | 32px | Layout gaps (`py-8`, `mb-8`) |
| 2xl | 48px | Not used in this phase |
| 3xl | 64px | Not used in this phase |

Justified exceptions:
- Toggle switch internal offsets (`after:top-[2px]`, `after:left-[2px]`) use 2px for sub-component positioning within the 20px (h-5) track. These are cosmetic interior offsets, not layout spacing, and are standard for toggle switch construction.

Source: Extracted from existing `DiscCard.tsx`, `SubmitForm.tsx`, `page.tsx` Tailwind classes.

## Typography

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Body | 14px (`text-sm`) | 400 (`font-normal`) | 1.5 (Tailwind default for `text-sm`) |
| Label | 14px (`text-sm`) | 400 (`font-normal`) | 1.5 |
| Heading | 18px (`text-lg`) | 700 (`font-bold`) | 1.4 |
| Display | 24px (`text-2xl`) | 700 (`font-bold`) | 1.33 |

2 weights only: 400 (normal) for body and label text, 700 (bold) for headings and display text. No intermediate weights (500, 600) are used in this phase.

Source: Existing `DiscDetailPage` uses `text-2xl font-bold` for page title, `text-lg` for section headings, `text-sm` for labels and body text. This phase normalizes to 2 weights.

## Color

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `#ffffff` / `#0a0a0a` (dark) | Page background, form surfaces |
| Secondary (30%) | `neutral-50` / `neutral-900` (dark) | Cards, preview boxes, fingerprint metadata bar, sibling disc cards |
| Accent (10%) | `blue-600` / `blue-500` (dark) | See reserved list below |
| Destructive | `red-700` / `red-300` (dark) | Error messages only |

Accent reserved for:
- Primary CTA button background (`bg-blue-600`)
- Interactive link text (`text-blue-600`)
- Input focus ring (`focus:border-blue-500 focus:ring-blue-500`)
- Brand mark in NavBar (`text-blue-600`)
- Current disc highlight border in sibling row (`border-blue-500`)

Additional semantic colors (existing, reuse as-is):
- Success: `green-700` / `green-300` (dark) -- success messages
- Warning/disputed: `amber-800` / `amber-200` (dark) -- disputed status badge
- Type badges: `blue-800`, `purple-800`, `green-800` -- content type chips (existing `TYPE_COLORS`)

Source: `globals.css` CSS custom properties, `DiscCard.tsx` badge colors, `SubmitForm.tsx` input/button classes, `DiscDetailPage` status badges.

## Component Inventory

### New Components

#### 1. `SiblingDiscs` (server component)

**Location:** `web/components/SiblingDiscs.tsx`
**Requirement:** SET-06
**Props:**
```typescript
interface SiblingDiscsProps {
  editionName: string | null;
  discNumber: number;
  totalDiscs: number;
  siblings: SiblingDiscSummary[];
  currentFingerprint: string;
}
```

**Layout:**
- Header line: "Part of: {edition_name} (Disc {N} of {total})" at `text-sm font-bold`
- If `edition_name` is null, header reads: "Part of a {total}-disc set (Disc {N} of {total})"
- Below header: horizontal scrollable row of sibling cards with `gap-4` (16px)
- Row uses `flex overflow-x-auto` for horizontal scroll on narrow viewports
- Row wraps to grid on wider viewports: `flex flex-wrap` above 768px

**Sibling Card Spec (each card):**
- Container: `rounded-lg border border-neutral-200 bg-neutral-50 p-4 min-w-[180px] max-w-[220px]` (dark: `border-neutral-800 bg-neutral-900`)
- Current disc card: `border-blue-500 bg-blue-50` (dark: `border-blue-400 bg-blue-950`)
- Disc number: `text-xs font-normal text-neutral-500` -- "Disc {N}"
- Format badge: `text-xs rounded px-2 py-1` using existing `TYPE_COLORS` pattern but for format (DVD/Blu-ray/UHD)
- Main title: `text-sm font-bold truncate` -- main feature title or "Untitled"
- Duration: `text-xs text-neutral-400` -- formatted as "2h 15m" or omitted if null
- Track count: `text-xs text-neutral-400` -- formatted as "{N} tracks" or omitted if null (per D-06)
- Entire card is a link to `/disc/{fingerprint}` via Next.js `Link`
- Cards for discs not yet in the database (empty slots): show "Disc {N}" with `text-neutral-400` and dashed border `border-dashed`

**Format Badge Colors:**

| Format | Light | Dark |
|--------|-------|------|
| DVD | `bg-amber-100 text-amber-800` | `bg-amber-900 text-amber-200` |
| Blu-ray | `bg-blue-100 text-blue-800` | `bg-blue-900 text-blue-200` |
| UHD | `bg-violet-100 text-violet-800` | `bg-violet-900 text-violet-200` |

**Placement on disc detail page:** Between the fingerprint metadata bar and the "Titles" section. Wrapped in a `<section className="mb-8">` with `data-testid="sibling-discs"`.

#### 2. `SetSearchInput` (client component)

**Location:** `web/components/SetSearchInput.tsx`
**Requirement:** SET-07
**Props:**
```typescript
interface SetSearchInputProps {
  onSelect: (setId: string, setInfo: DiscSetSearchResult) => void;
  onCreateNew: () => void;
}
```

**Behavior:**
- Text input with `debounce` of 300ms before firing API search
- Uses existing `inputClass` styling from SubmitForm (matches project pattern)
- Placeholder: "Search sets by release title or edition..."
- Below input: dropdown results list (absolute positioned, `z-10`)
- Each result row: `px-4 py-2 hover:bg-neutral-100 dark:hover:bg-neutral-800 cursor-pointer text-sm`
- Result row content: "{Release Title} -- {Edition Name} ({N}/{total_discs} discs linked)"
- Bottom of dropdown: "Create new set" action row in `text-blue-600 font-normal`
- Empty search state: no dropdown shown
- No results state: dropdown with single row: "No matching sets found" + "Create new set" action
- Loading state: single row with "Searching..." at `text-neutral-400`

**Dropdown container:** `absolute mt-1 w-full rounded-lg border border-neutral-200 bg-white shadow-lg max-h-60 overflow-y-auto dark:border-neutral-800 dark:bg-neutral-950`

### Modified Components

#### 3. `SubmitForm` extension

**Requirement:** SET-07
**Changes:**
- New toggle after "Disc Details" fieldset: "Part of a multi-disc set?" as a checkbox/toggle
- Toggle label: `text-sm font-normal` with `data-testid="set-toggle"`
- When toggle is ON, reveal a `<fieldset>` with `data-testid="set-fields"`:
  - `SetSearchInput` component for existing set search
  - OR "New Set" sub-form with:
    - Disc number input (reuse existing `disc-number` field, move into this section)
    - Total discs input (reuse existing `total-discs` field, move into this section)
    - Edition name input with autocomplete suggestions
- When toggle is OFF: disc_number defaults to 1, total_discs defaults to 1, no set fields shown
- Edition name autocomplete: static client-side list (D-11), rendered as datalist or custom dropdown

**Edition Suggestions (D-11):**
```
Extended Edition, Director's Cut, Theatrical, Criterion Collection, Special Edition, Ultimate Edition
```

**Toggle implementation:** Standard checkbox input styled as toggle:
```
<label className="relative inline-flex items-center cursor-pointer">
  <input type="checkbox" className="sr-only peer" />
  <div className="w-9 h-5 bg-neutral-200 peer-focus:ring-2 peer-focus:ring-blue-500 rounded-full peer peer-checked:bg-blue-600 after:content-[''] after:absolute after:top-[2px] after:left-[2px] after:bg-white after:rounded-full after:h-4 after:w-4 after:transition-all peer-checked:after:translate-x-full dark:bg-neutral-700"></div>
</label>
```

Note: `after:top-[2px]` and `after:left-[2px]` are justified exceptions (see Spacing Scale section).

#### 4. `DiscDetailPage` extension

**Requirement:** SET-06
**Changes:**
- Import and render `SiblingDiscs` when `disc.disc_set` is not null
- Placement: after fingerprint metadata bar, before Titles section
- Conditional: `{disc.disc_set && <SiblingDiscs ... />}`
- No changes to existing sections

## Interaction States

### Sibling Disc Cards

| State | Visual |
|-------|--------|
| Default | `border-neutral-200 bg-neutral-50` |
| Hover | `hover:border-neutral-300 hover:shadow-sm transition-all` |
| Current disc | `border-blue-500 bg-blue-50` (not clickable, no hover effect) |
| Empty slot | `border-dashed border-neutral-300 bg-transparent` with muted text |

### Set Toggle (Submit Form)

| State | Visual |
|-------|--------|
| Off | Toggle grey (`bg-neutral-200`), set fields hidden |
| On | Toggle blue (`bg-blue-600`), set fields animate in with height transition |
| Transition | `transition-all duration-200` on the revealed fieldset |

### Set Search Input

| State | Visual |
|-------|--------|
| Empty | No dropdown |
| Typing (< 300ms) | No dropdown (debounce pending) |
| Loading | Dropdown with "Searching..." row |
| Results | Dropdown with result rows |
| No results | Dropdown with "No matching sets found" + "Create new set" |
| Result selected | Input shows selected set info, dropdown closes |

### 409 Conflict (disc number taken)

| State | Visual |
|-------|--------|
| Error | Standard error banner: `border-red-200 bg-red-50 text-red-700` (matches existing `submitError` pattern) |

## Responsive Behavior

| Breakpoint | Sibling Cards Layout | Set Fields Layout |
|------------|---------------------|-------------------|
| < 640px (mobile) | Horizontal scroll, single row, `min-w-[180px]` per card | Stack vertically, full width |
| 640-1024px (tablet) | Wrap to 2-3 cards per row | Two-column grid for disc number + total discs |
| > 1024px (desktop) | Wrap to 4+ cards per row within `max-w-5xl` container | Two-column grid (matches existing form layout) |

Source: Existing page uses `mx-auto max-w-5xl px-4` container. Sibling cards fit within this.

## Accessibility

| Concern | Implementation |
|---------|---------------|
| Sibling card links | Each card is an `<a>` (via Next.js `Link`) with `aria-label="View Disc {N}: {main_title}"` |
| Current disc indicator | `aria-current="true"` on the current disc card, plus visual border |
| Set toggle | Standard checkbox with associated `<label>`, `role` not needed (native element) |
| Search dropdown | `role="listbox"` on dropdown, `role="option"` on each result row |
| Keyboard navigation | Arrow keys navigate dropdown results, Enter selects, Escape closes |
| Empty slot cards | `aria-disabled="true"`, not focusable (no href) |
