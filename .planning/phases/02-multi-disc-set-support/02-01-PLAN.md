## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-01 | Tampering | POST /v1/set | mitigate | Pydantic validates release_id (min_length=1), total_discs (ge=1). release_id checked against DB via FK query before use. Auth required via get_current_user Depends. |
| T-02-02 | Tampering | POST /v1/disc disc_set_id | mitigate | disc_set_id validated against DB (query DiscSet by ID, 404 if missing). disc_number validated <= total_discs (422). UniqueConstraint prevents duplicate disc_number (409). |
| T-02-03 | Information Disclosure | GET /v1/set/{id} | accept | Set data is public (read endpoints unauthenticated per API contract). No PII in set records. |

| T-02-05 | Tampering | GET /v1/set?q= injection | mitigate | SQLAlchemy ORM ilike() uses parameterized queries. No raw SQL. |
| T-02-06 | Denial of Service | disc_number squatting | accept | Rate limiting prevents bulk squatting. D-08 says first-come-first-served; users dispute via existing edit workflow. Low risk given auth requirement. |
