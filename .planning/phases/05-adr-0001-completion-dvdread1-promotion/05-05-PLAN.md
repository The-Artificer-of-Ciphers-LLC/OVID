<objective>
Add server-side hybrid primary selection (D-03): on a NEW disc submission, the server — not the client — chooses the primary among the submitted identity strings, preferring `dvdread1-*` whenever present. Wire this `_select_primary()` helper plus `register_fingerprint()` into both new-disc creation SAVEPOINTs (`register_disc`, `submit_disc`), and prove the mixed-fleet zero-fragmentation guarantee: an old client resubmitting `dvd1-*` as its declared primary can never demote an already-promoted disc.

Purpose: `Disc.fingerprint` is already structurally immutable on every existing-disc code path (confirmed by a full read of `_handle_existing_disc`/`_identify_existing_disc`/`_handle_existing_registered_disc` — none reassigns it) — this plan's real work is entirely in the NEW-disc creation branch's selection logic, and in proving the immutability invariant holds under a heterogeneous client fleet.
Output: `_select_primary()` colocated with `_method_of()`; both new-disc SAVEPOINTs updated to select the primary and register it in the WR-02 registry; a passing mixed-fleet regression test.
</objective>
