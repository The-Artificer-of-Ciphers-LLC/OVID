<objective>
Close the WR-02 cross-table fingerprint race (carried forward from the Phase 1 code review) by adding a new `fingerprint_registry` table with a single global `UNIQUE(fingerprint)` column (D-02), and wiring a `register_fingerprint()` helper into `attach_lookup_aliases()`'s existing SAVEPOINT so a "new disc claims F" vs. "attach F as an alias of a different disc" race collapses into the identical `IntegrityError`-catch/re-resolve/converge shape the codebase already handles for same-table races.

Purpose: promotion in this phase increases write concurrency on the shared fingerprint namespace (submissions now routinely carry both `dvd1-*` and `dvdread1-*` strings); without this registry, the `discs.fingerprint` and `disc_identity_aliases.fingerprint` UNIQUE constraints have no cross-table arbitration and a race between them can silently split one physical pressing into two disc rows.
Output: `FingerprintRegistry` ORM model, `register_fingerprint()` helper wired into the alias-attach path, and a passing WR-02 regression test proving the cross-table race is closed.
</objective>
