<objective>
Ship the D-05 one-command cutover wrapper (`scripts/promote_dvdread1.py`) that performs toggle-read-only -> `alembic upgrade head` -> toggle-read-write as a single operator action, reusing the existing, already-tested `MirrorModeMiddleware` (D-04) with zero new middleware code. Document the runbook in `docs/self-hosting.md` (with an explicit "this also interrupts reads" caveat, per Pitfall 3) and cross-reference it from `docs/deployment.md` for the canonical-server audience, who — unlike self-hosting.md's mirror-mode Quick Start — is NOT already permanently read-only and genuinely needs this wrapper.

Purpose: this closes the "operator forgets to flip writes back on" failure mode the research flagged, by making the mode-capture/restore behavior a scripted invariant rather than a manual multi-step runbook.
Output: `scripts/promote_dvdread1.py`; a new self-hosting.md section; a new deployment.md cross-reference section.
