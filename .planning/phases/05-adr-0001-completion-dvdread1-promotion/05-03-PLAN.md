<objective>
Flip `identify_dvd()` so `dvdread1-*` becomes the client-computed primary whenever libdvdread succeeds (with `dvd1-*` as the alias), mirroring `identify_bd()`'s already-established Tier-2-primary/Tier-1-alias pattern from Phase 4. When libdvdread is unavailable or returns an invalid Disc ID, `dvd1-*` remains primary with zero aliases — unchanged from today.

Purpose: this closes RESEARCH.md's Open Question #1 and is one half of the fix for the ARM auto-register gap (arm/identify_ovid.py sends only `disc.fingerprint` with zero aliases today — Plan 05-04 fixes ARM's payload directly, but this flip means `disc.fingerprint` itself is already the richer `dvdread1-*` string whenever computable, which helps ANY caller that only reads the bare fingerprint).
Output: `identify_dvd()` prefers `dvdread1-*` as primary whenever computable; the two existing failure-path tests are confirmed unchanged; the one success-path test is intentionally rewritten to assert the new behavior.
</objective>
