<objective>
Introduce the required `OVID_ENV` environment variable and its import-time boot assertion (D-09,
AUTH-10). Mirror the existing `_require_env("OVID_SECRET_KEY")` fail-fast in the SAME file
(`api/app/auth/config.py`). Derive a single `ALLOW_LOCALHOST_BYPASS` constant from `OVID_ENV` so the
IndieAuth localhost bypass is provably unreachable under `OVID_ENV=production`. Keep this guard in
config.py, evaluated unconditionally at import — never inside an IndieAuth-only branch (Pitfall 6).
Because config.py now requires OVID_ENV, the test conftest must set it before any app import, and all
deployment compose files + `.env.example` must declare it (a deliberate breaking change for existing
self-hosted instances, surfaced in docs by Plan 07).

Purpose: Make "someone forgot to flip the dev bypass off in prod" impossible — the app refuses to
boot until the operator declares its environment, and production can never enable the bypass.
Output: OVID_ENV boot assertion + ALLOW_LOCALHOST_BYPASS in config.py; conftest + compose + .env.example plumbing; test_auth_config.py.
</objective>
