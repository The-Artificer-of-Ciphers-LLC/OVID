Add the permanent anti-fragmentation CI regression test (IDENT-05, [guardrail]): `api/tests/test_disc_identity_regression.py`, an UNMARKED pytest that seeds a real `dvd1-`-prefixed OVID-DVD-1 pressing and asserts it still resolves through `GET /v1/disc` with frozen structure. Per D-13 use a golden ORM-seeded record asserted through the API (reuse the `seed_test_disc`/`conftest.py` fixture pattern) — NOT a synthetic fabricated row and NOT a JSON snapshot file (which invites "just re-approve" that swallows the regression).

Per RESEARCH Pattern 3: `seed_test_disc` seeds `fingerprint="dvd-ABC123-main"` (NOT a `dvd1-*` string), so write a dedicated golden seeder for a `dvd1-`-prefixed primary representing a genuine pre-migration identity. Per D-14 assert the resolved title/chapter/track/release fields against a HARDCODED expected dict written into the test (NOT re-read from the seed inputs — else the test is tautological); IDENT-05 catches silent data drift, not just a 200. Add a `# guardrail: IDENT-05` docstring for discoverability; it must NOT depend on any hardware fixture.

Purpose: A permanent, structure-freezing guard that catches any silent corruption of `dvd1-*` resolution — the invariant ADR 0001's stability guarantee needs, and the invariant tied to this phase's assumption-delta decision (Plan 01).
