| Boundary | Description |
|----------|-------------|
| client -> API POST /v1/disc | Untrusted chapter data in submit request body |
| API -> database | Chapter data written to disc_chapters table |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-01 | Tampering | ChapterCreate schema | mitigate | Pydantic validates chapter_index ge=1, name max_length=200, start_time_secs ge=0. SQLAlchemy ORM parameterized queries prevent injection. |
| T-03-02 | Denial of Service | submit_disc route | mitigate | Validate max 999 chapters per title in submit_disc before DB writes. Reject with 400 if exceeded. |
| T-03-03 | Spoofing | ChapterCreate.name | accept | Chapter names are user-contributed metadata. React auto-escapes on display. No HTML rendering of chapter names. |
| T-03-04 | Information Disclosure | ChapterResponse | accept | Chapter data is public metadata (CC0 licensed). No PII in chapter names. |
</threat_model>
