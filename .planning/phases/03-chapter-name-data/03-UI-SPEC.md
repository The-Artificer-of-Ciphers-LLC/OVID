## Component Inventory

### New Components

#### 1. `ChapterList` (client component)

**Location:** `web/components/ChapterList.tsx`
**Directive:** `"use client"` (requires `useState` for expand/collapse)
**Requirement:** CHAP-06
**Props:**
```typescript
interface ChapterListProps {
  chapters: ChapterResponse[];
  titleIndex: number;
}
```

**Layout:**
- Expand/collapse button: `text-sm text-blue-600 cursor-pointer` with text "{N} chapters" where N is `chapters.length`
- Collapsed by default (D-10)
- When expanded: a compact table below the title row with columns: #, Name, Start Time
- Table container: `ml-8 mt-2 mb-2` (indented 32px from left edge to visually nest under the parent title row)
- Table: `text-xs` for all content (one step smaller than the parent title table at `text-sm`)

**Chapter Row Spec:**
- Index column: `tabular-nums text-neutral-500 pr-3` width 32px -- displays 1-based chapter_index
- Name column: `pr-3` -- displays chapter name or em-dash (`\u2014`) when null
- Start time column: `tabular-nums text-neutral-400` -- formatted as `H:MM:SS` or `M:SS` (see Time Format below)

**Time Format (Claude's discretion decision):**
- Use `H:MM:SS` when hours > 0 (e.g., `1:23:45`)
- Use `M:SS` when hours == 0 (e.g., `3:45`)
- Display em-dash (`\u2014`) when `start_time_secs` is null
- Matches the existing `formatDuration` pattern in `DiscStructure.tsx` but without zero-padding the leading unit

**Expand/Collapse Animation (Claude's discretion decision):**
- No animation. Toggle visibility with conditional rendering (`{expanded && <table>...}</table>}`).
- Rationale: The project has no animation library. CSS height transitions on dynamic content require measuring, which adds complexity for no functional value. SiblingDiscs uses no animation either.

**Expand/Collapse Button Spec:**
- Text: "{N} chapters" (e.g., "12 chapters")
- Chevron: Unicode `\u25B6` (right-pointing triangle) when collapsed, `\u25BC` (down-pointing triangle) when expanded
- Full button: `<button className="text-sm text-blue-600 hover:underline cursor-pointer inline-flex items-center gap-1" data-testid="chapter-toggle-{titleIndex}">`
- Placement: Replaces the static chapter_count number in the Chapters column of DiscStructure

#### 2. `ChapterEditor` (client component)

**Location:** `web/components/ChapterEditor.tsx`
**Directive:** `"use client"` (manages chapter row state)
**Requirement:** CHAP-07
**Props:**
```typescript
interface ChapterEditorProps {
  titleIndex: number;
  chapters: ChapterCreate[];
  onChange: (chapters: ChapterCreate[]) => void;
}

interface ChapterCreate {
  chapter_index: number;
  name: string | null;
  start_time_secs: number | null;
}
```

**Layout:**
- Expandable section with header: "Add chapters" as a button toggle
- Header button: `text-sm text-blue-600 hover:underline cursor-pointer inline-flex items-center gap-1`
- Chevron: Same `\u25B6` / `\u25BC` pattern as ChapterList
- Collapsed by default (D-11)
- When expanded: a list of chapter input rows + action buttons

**Chapter Input Row Spec:**
- Container: `flex items-center gap-3 mb-2`
- Index display: `text-sm tabular-nums text-neutral-500 w-8 text-right` -- auto-increment, read-only, displays row position (1, 2, 3...)
- Name input: `flex-1` using project `inputClass` pattern (`rounded border border-neutral-300 bg-white px-3 py-2 text-sm outline-none focus:border-blue-500 focus:ring-1 focus:ring-blue-500 dark:border-neutral-700 dark:bg-neutral-900`)
  - Placeholder: "Chapter name (optional)"
  - `maxLength={200}` (D-05)
  - `data-testid="chapter-name-{titleIndex}-{chapterIndex}"`
- Start time input: `w-28` using project `inputClass` pattern
  - Placeholder: "0:00:00"
  - `type="text"` with pattern validation for `H:MM:SS` or `M:SS` format
  - `data-testid="chapter-time-{titleIndex}-{chapterIndex}"`
  - Parse to integer seconds on blur: split by `:`, compute `h*3600 + m*60 + s`
  - Invalid input: red border (`border-red-500`) with no error message (input clears on next focus)
- Remove button: `text-neutral-400 hover:text-red-600 text-sm cursor-pointer` displaying Unicode `\u00D7` (multiplication sign)
  - `data-testid="chapter-remove-{titleIndex}-{chapterIndex}"`

**Action Buttons:**
- "Add chapter" button: `text-sm text-blue-600 hover:underline cursor-pointer mt-1`
  - `data-testid="chapter-add-{titleIndex}"`
  - Appends a new row with `chapter_index = last + 1`, `name = null`, `start_time_secs = null`
- No "Remove all" button (remove individual rows one at a time)

**Tab Order (Claude's discretion decision):**
- Tab moves: Name input -> Time input -> Next row Name input -> Next row Time input
- Remove button is focusable but not in the natural tab flow (placed after time input)
- "Add chapter" button receives focus after the last row's time input

**Validation Timing (Claude's discretion decision):**
- Name: No validation beyond maxLength (browser-enforced)
- Start time: Validate format on blur. If invalid format, clear the field and set `start_time_secs = null`
- No blocking validation -- invalid times are silently cleared, not rejected

### Modified Components

#### 3. `DiscStructure` modification

**Requirement:** CHAP-06
**Current state:** Server component, renders title table with static chapter_count column
**Changes:**
- Convert to client component (`"use client"` directive) OR extract the chapter column cell into a `ChapterList` client component rendered within the server component table
- Recommended approach: Keep `DiscStructure` as a server component. Replace the static `{title.chapter_count ?? "\u2014"}` cell content with `<ChapterList chapters={title.chapters} titleIndex={title.title_index} />` which is a client component
- When `title.chapters` is empty or undefined: display em-dash (`\u2014`) as before (no expand button)
- When `title.chapters` has entries: display the `ChapterList` expand button

**ChapterList placement within table:**
- The expand button replaces the chapter count cell content
- The expanded chapter table renders in a new `<tr>` below the title row, spanning all 6 columns via `<td colSpan={6}>`

#### 4. `SubmitForm` modification

**Requirement:** CHAP-07
**Changes:**
- Add `ChapterEditor` section per title (currently titles array is empty `[]`, but the form should support manual title + chapter entry for future use)
- For Phase 3 scope: Add a single expandable "Chapters" section after the fingerprint preview
- This section contains one `ChapterEditor` per detected title from `fpData.structure`
- Each title section header: `text-sm font-medium` showing "Title {N}" with the `ChapterEditor` below
- Chapters array is included in the `titles` payload field

**Integration with submit payload:**
- The `SubmitForm` `handleSubmit` function maps chapter editor state into `TitleCreate.chapters` arrays
- Each `ChapterCreate` has: `chapter_index` (1-based), `name` (string or null), `start_time_secs` (integer or null)

#### 5. `api.ts` type extensions

**Requirement:** CHAP-04, CHAP-06
**Changes to `web/lib/api.ts`:**
```typescript
export interface ChapterResponse {
  chapter_index: number;
  name: string | null;
  start_time_secs: number | null;
}
```

Add to `TitleResponse`:
```typescript
chapters: ChapterResponse[];
```

Add to request types:
```typescript
export interface ChapterCreate {
  chapter_index: number;
  name: string | null;
  start_time_secs: number | null;
}
```

Add to `TitleCreate`:
```typescript
chapters?: ChapterCreate[];
```

## Interaction States

### Chapter List (Disc Detail)

| State | Visual |
|-------|--------|
| No chapters | Em-dash in chapter count column (same as current) |
| Collapsed (has chapters) | "{N} chapters" link with right-pointing triangle |
| Expanded | Down-pointing triangle, chapter table visible below title row |
| Hover on toggle | Underline on text |

### Chapter Editor (Submit Form)

| State | Visual |
|-------|--------|
| Collapsed | "Add chapters" with right-pointing triangle |
| Expanded, empty | "Add chapters" expanded, single "Add chapter" action link |
| Expanded, with rows | Chapter input rows with index, name, time, remove button |
| Invalid time input | Red border on time input (`border-red-500`), clears on blur |
| Row being removed | Row disappears immediately (no animation) |

### Chapter Time Input

| State | Visual |
|-------|--------|
| Empty | Placeholder "0:00:00" in `text-neutral-400` |
| Valid format | Normal border, value displayed |
| Invalid format (on blur) | Field clears, border returns to normal |
| Focused | Blue ring (`focus:border-blue-500 focus:ring-1 focus:ring-blue-500`) |
